# Contributing

Thank you for improving Guardian Vision.

## Development Workflow

1. Create a feature branch.
2. Keep changes focused.
3. Run lint and build before opening a pull request.
4. Update documentation when behavior changes.
5. Include screenshots for UI changes.

## Local Commands

```bash
npm install
npm --prefix server install
npm run build
npm run lint
```

Use `npm.cmd` on Windows PowerShell if script execution policy blocks `npm`.

## Pull Request Checklist

- The app builds.
- New environment variables are documented.
- Security-sensitive changes are explained.
- API changes are added to docs.
- UI changes include screenshots or a short demo note.
